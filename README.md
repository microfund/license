# GitHub主要ライセンス一覧（比較表）

| ライセンス名                         | 商用利用              | クレジット(著作権表示) | コピーレフト        | 特徴（要点）                                  |
| ------------------------------ | ----------------- | ------------ | ------------- | --------------------------------------- |
| **No license（無許諾）**            | ❌不可（デフォルトでは利用権なし） | –            | –             | 公開しても他人は自由に利用できない。著作権は保持されるだけで「許可」がない状態 |
| **Apache License 2.0**         | ✔️                | ✔️           | ❌（緩やか）        | 特許権の明確な許諾がある。企業向けに最も安全性の高いライセンス         |
| **GNU GPL v3.0**               | ✔️                | ✔️           | ✔️強いコピーレフト    | 派生ソフトも GPL で公開必須。オープンソースの強力な共有を強制       |
| **MIT License**                | ✔️                | ✔️           | ❌             | 最も緩いライセンス。自由に使えて、再配布も容易。商用もOK           |
| **BSD 2-Clause**               | ✔️                | ✔️           | ❌             | MITに近い非常に緩いライセンス。2つの条件（著作権表示と免責のみ）      |
| **BSD 3-Clause**               | ✔️                | ✔️           | ❌             | 上記に「名前の使用禁止条項」が追加。企業でよく採用される            |
| **Boost Software License 1.0** | ✔️                | ✔️           | ❌             | C++界で多用。MIT・BSDと同等で非常に緩い                |
| **Creative Commons Zero（CC0）** | ✔️                | 不要           | ❌             | パブリックドメイン相当。「完全に自由」なライセンス               |
| **Eclipse Public License 2.0** | ✔️                | ✔️           | ✔️弱いコピーレフト    | Eclipse 系プロジェクトで利用。モジュール単位のコピーレフト       |
| **GNU AGPL v3.0**              | ✔️                | ✔️           | ✔️非常に強いコピーレフト | Webサービスに適用。ネット越し利用でもソース開示義務             |
| **GNU GPL v2.0**               | ✔️                | ✔️           | ✔️            | 古い GPL。Linux カーネルが採用                    |
| **GNU LGPL v2.1**              | ✔️                | ✔️           | 部分的           | ライブラリ向け。利用側はGPLでなくてもよい                  |
| **Mozilla Public License 2.0** | ✔️                | ✔️           | ✔️弱いコピーレフト    | ファイル単位での開示義務。商用にも柔軟                     |
| **The Unlicense**              | ✔️                | 不要           | ❌             | パブリックドメイン宣言。CC0に似て完全自由                  |

# 用語の簡単説明

| 用語           | 説明                             |
| ------------ | ------------------------------ |
| 商用利用         | お金を稼ぐ目的の利用が可能か                 |
| 著作権表示（クレジット） | コピーや改変して再配布する際に著作権者名を残す必要があるか  |
| コピーレフト       | 派生物も同じライセンスで公開する義務             |
| 強いコピーレフト     | すべての派生物を公開しなければならない（GPL, AGPL） |
| 弱いコピーレフト     | 対象ファイルやモジュールに限定（MPL, LGPL）     |

# 用途別おすすめ
| 目的                  | 最適なライセンス                  |
| ------------------- | ------------------------- |
| 商用・クローズドソースで使ってほしい  | **MIT / BSD / Apache2.0** |
| オープンソースで派生物も公開してほしい | **GPL / AGPL**            |
| コードを完全に自由に解放したい     | **CC0 / Unlicense**       |
| 特許の保護も欲しい           | **Apache 2.0**            |
| Webサービスにも共有義務を課したい  | **AGPL**                  |
